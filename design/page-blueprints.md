# Page Blueprints

## Landing Page Blueprint

### Goal

Make the site feel like a guided operating system within the first screen and first scroll.

### Section Order

1. Hero
2. Belief system
3. Guided bottleneck selector
4. Framework preview cards
5. How the system works
6. Proof through usefulness
7. Closing CTA

### Hero Layout

- large serif headline on the left
- short subhead underneath
- primary and secondary CTAs grouped tightly
- utility rail or side note on the right with:
  - audience
  - promise
  - first action

This section should feel bold and architectural, not like a centered SaaS hero.

### Belief System Layout

Render beliefs as punchy statement rows, not body paragraphs.

Use:

- a section title
- five short principle blocks
- each block with a number, short label, and one-line claim

### Guided Bottleneck Selector

Make this a high-utility module.

Structure:

- title: `Choose Your Bottleneck`
- three large selectable lanes:
  - strategy
  - prioritization
  - metrics
- each lane shows:
  - symptom
  - what the playbook helps with
  - direct CTA

### Framework Preview Cards

Each card should show both layers:

- framework summary
- operator attachment preview

Card structure:

- category label
- problem line
- when to use it
- primary CTA: framework
- secondary CTA: operator mode

### How The System Works

Present as a three-step horizontal process on desktop and stacked timeline on mobile.

- find the bottleneck
- read the framework
- switch to operator mode and run it live

### Closing CTA

Repeat the core value proposition with tighter copy and one clear CTA.

## Framework Index Blueprint

### Goal

Act as a decision navigator, not a content archive.

### Section Order

1. Intro summary band
2. Decision-path prompt
3. Framework cards
4. Operator mode explainer

### Intro Summary Band

Include:

- short headline
- one-sentence promise
- scan hint such as `Start with the bottleneck you feel right now`

### Decision-Path Prompt

Show a short chooser:

- no clear direction -> strategy
- too many competing priorities -> prioritization
- lots of numbers, low clarity -> metrics

### Framework Cards

Each card should be wider than a typical blog card and more structured.

Use rows or large blocks, not tiny tiles.

Required content:

- problem
- when to use it
- what you get
- framework CTA
- operator CTA

## Framework Page Blueprint

### Goal

Let readers absorb the thinking fast, then deliberately move deeper.

### Page Frame

Use a split layout:

- main reading column
- sticky utility rail

Utility rail contains:

- page summary
- key CTA
- mode switch
- checklist jump links

### Section Order

1. Summary band
2. Problem
3. When to use it
4. Operating principle
5. Framework steps
6. Example
7. Worksheet or checklist
8. Operator attachment

### Summary Band

This should be immediately skimmable.

Include:

- one-sentence claim
- who it is for
- what it helps decide
- CTA to read or switch mode

### Framework Steps

Render each step as its own module with:

- step number
- action title
- short explanation
- prompt or decision question

### Example Block

Use a before-and-after presentation:

- before: vague or flawed operating state
- after: sharper bet or clearer decision state

### Operator Attachment

This should not look like a random related link.

Present it as a bordered or tinted action panel:

- title: `Switch To Operator Mode`
- one-sentence description
- short list of what the tactical playbook includes
- CTA

## Operator Playbook Blueprint

### Goal

Make the tactical layer feel procedural, practical, and immediately runnable.

### Section Order

1. Purpose summary band
2. Run this when
3. Who should be involved
4. Inputs required
5. Expected output
6. Workflow
7. Failure modes
8. Done standard

### Layout Rules

- denser module spacing than framework pages
- stronger borders and panel separation
- labels feel operational, not editorial
- workflow steps should visually resemble a runbook

### Workflow Modules

Each step should include:

- action
- prompt
- rule
- optional capture field

### Utility Enhancements

Good additions if the live site supports them:

- sticky step index
- print-friendly mode
- copyable checklist or template blocks

## Mobile Blueprint

- Hero keeps one bold headline and one compact CTA group above the fold.
- Utility rails collapse into summary cards below the header.
- Segmented mode controls remain visible near the top of framework pages.
- Framework cards become stacked blocks with clear CTA separation.
- Worksheet and checklist modules remain visually distinct from body text.

# Module Library

## Summary Band

### Purpose

Give readers the page in one glance.

### Contains

- category label
- headline
- one-sentence promise
- audience or use case
- primary CTA
- optional secondary CTA

## Principle Callout

### Purpose

Highlight a sharp point of view without burying it in paragraphs.

### Contains

- short label
- one strong sentence

### Visual Notes

- large serif quote feel
- thin rule above or below

## Bottleneck Selector

### Purpose

Help the user self-sort into strategy, prioritization, or metrics quickly.

### Contains

- symptom
- playbook match
- CTA

## Framework Card

### Purpose

Preview the framework and the operator layer together.

### Contains

- topic label
- problem
- when to use it
- what you get
- primary CTA
- secondary operator CTA

## Use-This-When Pills

### Purpose

Compress situational cues into highly scannable signals.

### Contains

- short, concrete triggers

### Visual Notes

- avoid overly rounded SaaS pill styling
- use sharper corners or understated chips

## Step Module

### Purpose

Turn process into readable, repeatable sequence.

### Contains

- step number
- title
- explanation
- prompt
- rule

## Example Block

### Purpose

Make the framework feel grounded and credible.

### Contains

- before state
- after state
- what changed

## Worksheet Panel

### Purpose

Separate practical tools from reading content.

### Contains

- panel title
- fill-in fields or checklist items
- optional copy/export affordance

## Operator Mode Panel

### Purpose

Bridge from framework reading into execution mode.

### Contains

- title
- tactical promise
- included items
- CTA

### Visual Notes

- stronger border than editorial modules
- slightly darker or denser background
- clear icon or label distinction from the framework layer

## Utility Rail

### Purpose

Keep navigation and action available without cluttering the content column.

### Contains

- page summary
- section links
- mode switch
- worksheet jump

## Recommended Visual Consistency Rules

- Every page begins with a summary band.
- Every framework page includes at least one principle callout, one example block, and one operator mode panel.
- Every operator playbook uses the same step module structure.
- Every CTA pair distinguishes between reading and running:
  - `Read the Framework`
  - `Switch to Operator Mode`
